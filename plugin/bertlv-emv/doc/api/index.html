bertlv_emv

Flutter plugin for parsing and explaining EMV BER-TLV data using Kotlin/Native + dart:ffi.

The Kotlin Multiplatform bertlv-emv library is compiled to native shared libraries (.so for Android, .framework for iOS) and called directly via dart:ffi — no method channels, no async overhead.

Platform Support

Platform Status Library
Android arm64-v8a, x86_64, armeabi-v7a libbertlv_emv.so
iOS arm64, x64, simulator-arm64 bertlv_emv.framework

Usage

import 'package:bertlv_emv/bertlv_emv.dart';

final emv = BertlvEmv();

// Get library version
print(emv.version);

// Parse a TLV list (e.g., TVR + CID + AIP)
final results = emv.parseTlvList('9505040000009F2701809F360200FF');
for (final tlv in results) {
  print('${tlv.tagHex} ${tlv.tagName}: ${tlv.valueHex}');
  if (tlv.explanation != null) {
    print(tlv.explanation);
  }
}

// Parse a single TLV
final tvr = emv.parseTlv('95050400000000');
print(tvr.explanation); // Multi-line bitfield breakdown

// Explain a tag value directly
final explanation = emv.explainTag('9F27', '80');
print(explanation); // JSON map with explanation details

API

BertlvEmv

Method Returns Description
version String Native library version
parseTlvList(hex) List<TlvResult> Parse hex string containing one or more TLVs
parseTlv(hex) TlvResult Parse a single TLV element
explainTag(tagHex, valueHex) Map<String, dynamic>? Explain a tag value (null if no explainer)

All methods are synchronous — no Future, no await.

TlvResult

Property Type Description
tagHex String Tag as hex (e.g., "9F26")
tagName String? Human-readable name
description String? EMV specification description
length int Value length in bytes
valueHex String Raw value as hex
isConstructed bool Whether tag is a container
children List<TlvResult>? Nested TLVs for constructed tags
explanation String? Formatted explanation (bitfield tables, lookups, etc.)

Building Native Libraries

# From project root:
make android        # Full chain: native build → copy → ffigen → flutter build
make native-android # Build .so files only
make ffigen         # Regenerate Dart FFI bindings

# Or manually:
cd lib && ./gradlew copyAndroidNativeLibs

See the project Makefile for all build targets.

Architecture

commonMain (Kotlin) — pure business logic
       ↓
  Kotlin/Native
       ↓
  C-API (CApi.kt, @CName exports)
       ↓
  .so / .framework
       ↓
  dart:ffi (DynamicLibrary.open)
       ↓
  BertlvEmv Dart class
